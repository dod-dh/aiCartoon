# 대본 포맷 (`scripts/<id>.md`)

사람이 작성하는 **입력 파일**. 가볍게 비트만 적으면 파이프라인이 컷 콘티로 확장한다.

## 구조

```markdown
---
id: ep1
title: 카페 오늘, 첫 손님
stage: 첫 만남
cast: [momo, mumu]
panels: 6
mood: 50현실50개그
hairPhase: early
---

## 로그라인
한 문장 요약.

## 비트
- 컷으로 확장될 사건들을 한 줄에 하나씩.

## 메모 (선택)
- 톤/필수 대사/금지사항 등 자유 지시.
```

## 프론트매터 필드

| 필드 | 필수 | 설명 |
|------|:---:|------|
| `id` | ✅ | 에피소드 식별자. 파일명과 일치 (`ep1` → `ep1.md`) |
| `title` | ✅ | 에피소드 제목 |
| `stage` | | 연애 진행 단계. `characters.json > relationship.stages` 중 하나 |
| `cast` | ✅ | 등장인물 **키** 배열. `momo` / `mumu` / `gureum` |
| `panels` | | 목표 컷 수(4~8). 생략 시 콘티 단계에서 자동 결정 |
| `mood` | | 톤 힌트(기본 50% 현실 + 50% 개그) |
| `hairPhase` | | 무무 헤어: `early`(댄디컷) / `late`(리프컷) |

## 섹션

- **`## 로그라인`** — 에피소드 한 줄 요약. 콘티 방향의 기준.
- **`## 비트`** — 불릿 목록. 한 줄이 대략 한 컷~두 컷으로 확장된다. `panels` 수에 맞춰 병합/분할됨.
- **`## 메모`** *(선택)* — 반드시 넣을 대사, 마무리 톤, 금지사항 등. 콘티 작가가 우선 반영.

## 캐릭터 키 ↔ 이름

| 키 | 이름 | 비고 |
|----|------|------|
| `momo` | 모모 | 여주 · ♡ 연핑크 포인트 |
| `mumu` | 무무 | 남주 · ★ 연블루 포인트 |
| `gureum` | 구름이 | 8살 진도믹스 반려견 |

> 대사 말투는 `characters.json > characters.*.speech` 를 절대 기준으로 삼는다.
> 비트에는 말투를 직접 안 써도 되고, 콘티 단계에서 캐릭터별 말투로 대사가 붙는다.

## 산출물 (참고)

콘티는 `outputs/<id>/storyboard.json` 에 아래 형태로 저장된다:

```json
{
  "episodeId": "ep1",
  "title": "카페 오늘, 첫 손님",
  "panels": [
    {
      "panel": 1,
      "characters": ["mumu"],
      "action": "무무가 카페 문을 열고 밝게 들어온다",
      "emotion": { "mumu": "웃음" },
      "dialogue": [{ "who": "mumu", "text": "안녕하세요!" }],
      "sfx": "딸랑"
    }
  ]
}
```
